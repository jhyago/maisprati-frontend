A Caixa

Box Model a fundo

Você já viu as quatro camadas no CSS Básico — agora é hora de dominar a caixa de verdade: o cálculo do tamanho real, o reset universal de box-sizing, o misterioso colapso de margens e o que acontece quando o conteúdo transborda.

02

O cálculo do tamanho real

box-sizing: border-box

No modo padrão (content-box), o width que você escreve mede só o conteúdo. O espaço real ocupado é a soma de tudo:

a conta do content-box 
.card {
  width: 300px;
  padding: 20px;
  border: 2px solid;
}
/* largura real = 300 + 20+20 (padding) + 2+2 (border) = 344px
   o margin empurra os vizinhos, mas não entra na largura da caixa */

Com box-sizing: border-box, a conta inverte: o width passa a ser o tamanho final da caixa, e o navegador espreme padding e borda para dentro dele. Por isso praticamente todo projeto começa com o reset universal:

o reset que todo projeto usa 
*,
*::before,
*::after {
  box-sizing: border-box;   /* width = tamanho final, sempre */
}
💡 Por que border-box venceu Com border-box, "quero um card de 300px" vira simplesmente width: 300px — ajuste o padding à vontade que a caixa não muda de tamanho. Layouts em colunas (width: 50% + padding) só funcionam sem dor de cabeça nesse modo.
03

Colapso de margens

1 + 1 = 1?

Surpresa clássica do CSS: quando duas margens verticais se encontram, elas não se somam — a maior "engole" a menor. Um parágrafo com margin-bottom: 24px seguido de outro com margin-top: 16px fica a 24px de distância, não 40px:

as margens colapsam: vale a maior (24px), não a soma (40px)

margin-bottom: 24px ↓

↑ margin-top: 16px — distância real: 24px

Pontos para guardar sobre o colapso:

SituaçãoColapsa?
Margens verticais de irmãos (um sobre o outro)✅ Sim — vale a maior
Margens horizontais (lado a lado)❌ Nunca — sempre somam
Pai sem padding/border e o primeiro filho✅ Sim — a margem do filho "vaza" para fora do pai
Itens dentro de display: flex ou grid❌ Não — flex/grid desligam o colapso
💡 Como evitar a briga de margens A convenção mais comum: defina a margem em uma direção só (por exemplo, todos os elementos usam apenas margin-bottom). E quando os elementos estão num container flex, prefira gap — espaçamento sem nenhum colapso. O gap você domina na página de Flexbox.
04

Overflow — quando o conteúdo não cabe

visible · hidden · scroll · auto

A caixa tem tamanho definido, mas o conteúdo é maior. E agora? A propriedade overflow decide:

a mesma caixa de 90px de altura com cada overflow
visible (padrão)
O texto vaza para fora da caixa e pode atropelar o vizinho de baixo. Quase nunca é o que você quer.
hidden
O que não cabe é simplesmente cortado, sem aviso. Bom para decorações; cuidado com texto.
auto
A barra de rolagem aparece só quando precisa. É a escolha segura para áreas de conteúdo dinâmico.
📜 visible · hidden · scroll · auto visible deixa vazar (padrão); hidden corta; scroll mostra barras de rolagem sempre (mesmo sem precisar); auto mostra só quando precisa — na dúvida, use auto.
⚠️ Borda ocupa espaço, outline não Prima da borda, a outline é desenhada por cima da caixa sem entrar no cálculo de tamanho — por isso é o que o navegador usa no anel de foco (:focus) de botões e inputs. Nunca remova o outline sem dar um substituto visível: é ele que permite navegar com o teclado.

Pratique

fixe o conteúdo

Pense na resposta antes de revelar.

1A conta da caixa ◆ Fácil

Sem box-sizing declarado, qual a largura real de .foto { width: 250px; padding: 15px; border: 5px solid; margin: 10px; }? E com box-sizing: border-box?

290px e 250px. No padrão content-box: 250 + 15×2 + 5×2 = 290px. Com border-box, os 250px já incluem padding e borda. O margin nunca entra na largura da caixa — ele só afasta os vizinhos (totalizando 310px e 270px de espaço ocupado).

2O espaço que sumiu ◆ Médio

Dois cards empilhados têm margin: 20px 0 cada. Você esperava 40px entre eles, mas o DevTools mostra 20px. O que está acontecendo e cite duas formas de obter os 40px.

É o colapso de margens: margens verticais de irmãos não somam, vale a maior (20px). Para ter 40px: (1) usar margin-bottom: 40px numa direção só; ou (2) colocar os cards num container display: flex; flex-direction: column; com gap: 40px — flex desliga o colapso.

3A margem que vazou ◆ Difícil

Você dá margin-top: 32px ao <h2> que é o primeiro filho de uma <section> — mas em vez de empurrar o título para baixo dentro da section, a section inteira desce 32px na página. Por quê, e como prender a margem dentro dela?

Colapso entre pai e primeiro filho: como a section não tem padding nem borda no topo, a margem do h2 "vaza" e vira margem da section. Qualquer barreira entre os dois resolve:

resposta_3
/* opção 1: respiro interno na section (a mais comum) */
section { padding-top: 1px; }     /* ou um padding de verdade */

/* opção 2: trocar a margem do filho por padding do pai */
section { padding-top: 32px; }
h2      { margin-top: 0; }
