JavaScript no Navegador

DOM & BOM

O HTML monta a página uma vez; o JavaScript a torna viva. O DOM (Document Object Model) é a árvore de objetos que representa o seu HTML — é por ela que você lê, muda, cria e remove elementos em tempo real. O BOM (Browser Object Model) é tudo que está em volta da página: a janela, a URL, o histórico e o armazenamento local.

01

O que é o DOM

document · árvore de nós

Quando o navegador carrega uma página, ele lê o HTML e constrói uma árvore de objetos na memória. Cada tag vira um nó (node); as tags aninhadas viram nós filhos. Essa árvore é o DOM, e o ponto de entrada para ela é o objeto global document.

o HTML… 
<body>
  <h1>Olá</h1>
  <ul>
    <li>Item</li>
  </ul>
</body>

…vira esta árvore, que o JavaScript enxerga e pode percorrer:

a árvore do DOM 
document
└─ html
   └─ body
      ├─ h1  → "Olá"
      └─ ul
         └─ li → "Item"
📌 HTML ≠ DOM O HTML é o texto inicial que você escreveu no arquivo. O DOM é a estrutura viva em memória — ela começa igual ao HTML, mas muda conforme o JavaScript a manipula. Quando você inspeciona o "Elements" no DevTools, está vendo o DOM atual, não o HTML original.
02

Selecionar elementos

querySelector · getElementById

Antes de mexer em um elemento, você precisa encontrá-lo na árvore. Os métodos modernos querySelector e querySelectorAll usam a mesma sintaxe de seletores do CSS — o que você já aprendeu lá vale aqui.

MétodoRetornaExemplo
getElementById('x')O elemento com id="x" (ou null)document.getElementById('titulo')
querySelector('sel')O primeiro que casa o seletor CSSdocument.querySelector('.card')
querySelectorAll('sel')Todos que casam (uma NodeList)document.querySelectorAll('li')
selecionando 
const titulo = document.querySelector('h1');     // primeira <h1>
const itens  = document.querySelectorAll('li');  // lista de todos os <li>

console.log(titulo.textContent);  // o texto da h1
console.log(itens.length);        // quantos li existem

// querySelectorAll devolve algo iterável — dá pra usar forEach:
itens.forEach((li, i) => console.log(i, li.textContent));
⚠️ querySelectorAll não é um Array de verdade Ele devolve uma NodeList. Ela tem forEach, mas não tem map/filter. Quando precisar desses, converta primeiro: [...document.querySelectorAll('li')] ou Array.from(...).
03

Ler e mudar conteúdo, classes e estilo

textContent · classList · style

Com o elemento em mãos, você lê e altera suas propriedades. As três mais usadas: o texto, as classes (para ligar/desligar estilos do CSS) e o estilo inline.

PropriedadePara quê
el.textContentLer/trocar só o texto (seguro — não interpreta HTML)
el.innerHTMLLer/trocar o HTML interno (poderoso e perigoso — veja o aviso)
el.classList.add/remove/toggleLigar/desligar classes CSS
el.setAttribute('href', …)Mudar qualquer atributo
el.style.color = 'red'Aplicar um estilo inline pontual
mudando o elemento 
const el = document.querySelector('#aviso');

el.textContent = 'Texto novo';        // troca o texto
el.classList.add('destaque');         // liga a classe .destaque do CSS
el.classList.toggle('aberto');        // liga se estava desligada e vice-versa
el.style.color = 'crimson';           // estilo inline pontual
el.setAttribute('title', 'Dica!');    // qualquer atributo
🚨 innerHTML com dados do usuário = risco de XSS innerHTML interpreta o texto como HTML. Se você jogar ali algo digitado por um usuário, um <script> escondido pode rodar na sua página. Para mostrar texto, prefira sempre textContent. Use innerHTML só com HTML que você controla.
04

Criar, inserir e remover nós

createElement · append · remove

O DOM não é só leitura: você cria elementos do zero e os encaixa na árvore. O fluxo é sempre o mesmo: 1) crie o nó, 2) configure-o, 3) anexe-o a um pai.

criando do zero 
const lista = document.querySelector('ul');

const li = document.createElement('li');  // 1) cria o nó (ainda solto)
li.textContent = 'Novo item';             // 2) configura
lista.append(li);                         // 3) anexa como último filho

li.remove();                              // e para remover, basta isto

Veja funcionando de verdade — este mini "to-do" cria e remove <li> reais no DOM desta página:

to-do ao vivo
    💡 Performance: cuidado com loops grandes Cada append pode forçar o navegador a recalcular o layout. Ao inserir muitos itens de uma vez, monte tudo num DocumentFragment e anexe-o uma vez só — o navegador desenha apenas no final.
    05

    Eventos

    addEventListener · click · input

    Páginas interativas reagem ao usuário: um clique, uma tecla, o envio de um formulário. Você "escuta" esses acontecimentos com addEventListener('tipo', função). A função recebe um objeto event com detalhes do que aconteceu.

    ouvindo eventos 
    const botao = document.querySelector('#meu-botao');

botao.addEventListener('click', (event) => {
  console.log('Clicaram em', event.target);
});

// Em formulários, evite o recarregamento padrão da página:
form.addEventListener('submit', (e) => {
  e.preventDefault();   // cancela o comportamento nativo
  // …seu código aqui
});
    contador de cliques ao vivo
    🎯 Delegação de eventos Em vez de pôr um listener em cada <li>, ponha um só no <ul> pai e descubra quem foi clicado por event.target. Como os eventos "sobem" pela árvore (bubbling), o pai escuta os cliques dos filhos — inclusive os criados depois. Menos listeners, mais performance.
    06

    BOM — o navegador em volta da página

    window · location · localStorage

    Se o DOM é a página, o BOM é o navegador ao redor dela. O objeto global é a window — tudo que é global vive nela (inclusive o document). Dela saem objetos úteis para tarefas do dia a dia:

    window.location
    a URL atual; redirecionar a página
    window.history
    voltar/avançar na navegação
    window.navigator
    dados do browser e do dispositivo
    localStorage
    guardar dados que sobrevivem ao reload
    BOM na prática 
    console.log(window.location.href);     // URL completa da página
// window.location.href = '/index.html'; // navega para outra página
// window.history.back();                 // volta uma página

// localStorage só guarda STRINGS — serialize objetos com JSON:
localStorage.setItem('tema', 'dark');
const tema = localStorage.getItem('tema');   // 'dark'

localStorage.setItem('user', JSON.stringify({ nome: 'Ana', nivel: 3 }));
const user = JSON.parse(localStorage.getItem('user'));   // objeto de volta

// Temporizadores também fazem parte do BOM:
setTimeout(() => console.log('1 segundo depois'), 1000);
    💡 Você já usou o BOM neste site O botão de tema claro/escuro (☀️/🌙) salva sua escolha em localStorage e a reaplica a cada visita. É exatamente o padrão setItem / getItem acima.

    1textContent ou innerHTML? ◆ Fácil

    Você vai mostrar na tela um comentário digitado por um usuário. Qual propriedade usar para escrever esse texto num elemento, e por quê?

    textContent. Ele trata o valor como texto puro, então um <script> escondido no comentário não roda. Usar innerHTML com dado de usuário abre brecha de XSS.

    2Pinte os pares ◆ Médio

    Complete o código para que cada <li> de índice par da lista receba a classe par. Use querySelectorAll e classList.

    solução
    itens.forEach((li, i) => {
  if (i % 2 === 0) li.classList.add('par');
});
    3Lembrar do usuário ◆ Difícil

    O localStorage só guarda strings, então serialize com JSON.stringify ao gravar e JSON.parse ao ler:

    resposta_3

const dados = JSON.parse(localStorage.getItem('dados'));   // objeto de volta
