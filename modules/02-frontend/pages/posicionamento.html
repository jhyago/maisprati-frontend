Camadas & Fluxo

Posicionamento a fundo

O CSS Básico apresentou os cinco valores de position — aqui você os domina na prática: quem é a referência de cada um, como o z-index realmente decide quem fica na frente, e as três construções que aparecem em todo site: navbar fixa, badge sobre card e modal centralizado.

01

O fluxo e os cinco valores

static · relative · absolute · fixed · sticky

Sem CSS de posicionamento, o navegador empilha as caixas na ordem do HTML — o fluxo normal. Os valores de position são formas de negociar com esse fluxo. A pergunta-chave para cada um é: "a caixa sai do fluxo? E quem é a referência do top/left?" Reveja no visualizador (role a moldura para sentir sticky e absolute):

positionSai do fluxo?ReferênciaRola com a página?
staticNão— (offsets ignorados)Sim
relativeNão (o espaço fica reservado)A própria posição originalSim
absoluteSimAncestral posicionado mais próximoJunto com o ancestral
fixedSimA janela (viewport)Não — fica parado
stickyNão, até grudarO container que rolaSim, até atingir o offset
🧭 "Ancestral posicionado" — a definição exata É qualquer ancestral com position diferente de static. O absolute sobe na árvore procurando o primeiro assim; se não achar nenhum, ancora no documento inteiro. Dar position: relative a um container (sem offsets) é a forma padrão de dizer: "os absolutes daqui de dentro se ancoram em mim".
02

z-index e o contexto de empilhamento

quem fica na frente?

Quando caixas se sobrepõem, o z-index decide a ordem das camadas — maior valor fica na frente. Duas regras que pegam todo mundo: ele só funciona em elementos posicionados (qualquer position ≠ static), e ele compete dentro do mesmo contexto de empilhamento:

três cartas posicionadas com z-index
z-index: 1
z-index: 2
z-index: 3
⚠️ O mistério do "z-index: 9999 que não funciona" Certas propriedades (como opacity menor que 1, transform, filter, ou um pai posicionado com z-index definido) criam um contexto de empilhamento — uma "bolha" de camadas. Filhos competem só dentro da bolha: um z-index: 9999 dentro de uma bolha que vale 1 perde para qualquer caixa de fora com z-index: 2. Quando um z-index gigante não resolver, procure no DevTools qual ancestral criou a bolha.
💡 Escala de camadas com sanidade Em vez de uma corrida armamentista (999, 9999, 99999…), projetos maduros definem uma escala: 1–10 elementos comuns, 100 dropdowns, 1000 modais. Poucos valores, com significado.
03

As três receitas de todo site

navbar · badge · modal

1 — Navbar que acompanha a rolagem. sticky quando ela faz parte do fluxo (não cobre nada ao carregar); fixed quando deve flutuar sempre por cima:

navbar 
/* opção A — sticky: ocupa o lugar no topo e gruda ao rolar */
.navbar {
  position: sticky;
  top: 0;              /* sem um offset, sticky não gruda! */
  z-index: 100;
}

/* opção B — fixed: flutua sobre o conteúdo desde o início */
.navbar {
  position: fixed;
  top: 0; left: 0; right: 0;
  z-index: 100;
}
body { padding-top: 64px; }  /* compensa a altura, senão a navbar cobre o conteúdo */

2 — Badge no canto do card. O combo relative (pai) + absolute (filho) que você viu no básico — agora entendendo o porquê: o pai vira o ancestral posicionado que ancora o filho.

3 — Modal centralizado. fixed cobrindo a tela toda (o fundo escurecido) + flex para centralizar a caixa:

modal 
.overlay {
  position: fixed;
  inset: 0;                      /* atalho para top/right/bottom/left: 0 */
  background: rgba(0, 0, 0, 0.6);
  z-index: 1000;

  display: flex;                 /* centraliza o filho .modal */
  align-items: center;
  justify-content: center;
}
.modal { background: #fff; border-radius: 12px; padding: 24px; }

💡 inset: 0 É o atalho moderno para top: 0; right: 0; bottom: 0; left: 0; — quatro declarações em uma. Combinado com position: fixed ou absolute, estica o elemento até as quatro bordas da referência.
04

Pratique

fixe o conteúdo

Pense na resposta antes de revelar.

1sticky que não gruda ◆ Fácil

Você escreveu .titulo { position: sticky; } e nada aconteceu ao rolar. O que está faltando?

Falta o offset: sticky precisa saber onde grudar — top: 0, por exemplo. Sem um lado definido, ele se comporta como static. (E o elemento também não pode estar preso dentro de um ancestral com overflow: hidden.)

2fixed ou sticky? ◆ Médio

Com position: fixed na navbar, o topo do conteúdo da página some atrás dela. Com sticky, isso não acontece. Explique a diferença de comportamento.

fixed sai do fluxo: a navbar deixa de ocupar espaço e o conteúdo sobe para o lugar dela — por isso a compensação manual (padding-top no body). sticky permanece no fluxo: ela ocupa o próprio espaço no topo e só "gruda" quando a rolagem a alcançaria — nada fica coberto ao carregar.

3O z-index gigante que perde ◆ Difícil

Um tooltip com position: absolute; z-index: 9999 aparece atrás de um banner que tem apenas z-index: 2. O pai do tooltip tem position: relative; z-index: 1;. Por que o 9999 perde para o 2, e como corrigir?

O pai (posicionado e com z-index: 1) criou um contexto de empilhamento. O tooltip compete com o mundo externo "encapsulado" no valor do pai: 1. Como 1 < 2, o banner vence — o 9999 só vale dentro da bolha. Correções: subir o z-index do pai (ex.: 10), ou remover o z-index: 1 do pai para não criar a bolha:

resposta_3
.pai     { position: relative; }   /* sem z-index → sem bolha */
.tooltip { position: absolute; z-index: 10; }
