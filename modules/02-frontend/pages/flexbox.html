Layout Moderno

Flexbox

Alinhar caixas era a maior dor do CSS — até o display: flex. Com uma linha, o container ganha superpoderes para distribuir, alinhar e redimensionar os filhos. Aqui você aprende os dois eixos, as propriedades do container e dos itens, e as receitas que resolvem 90% dos layouts.

01

Container, itens e os dois eixos

main axis · cross axis

O Flexbox tem dois papéis: o container (quem recebe display: flex) e os itens (os filhos diretos dele). O container organiza os itens ao longo de dois eixos:

eixo principal (main)
a direção do flex-direction — onde os itens se enfileiram
eixo cruzado (cross)
o perpendicular — onde os itens se alinham na "altura" da linha
flex-direction: row → main = horizontal, cross = vertical
main axis ⟶ cross axis ⟶
item 1
item 2
item 3
🧭 A chave para nunca mais confundir justify e align justify-content distribui no eixo principal; align-items alinha no eixo cruzado. Quando flex-direction: column, os eixos giram — e o justify passa a ser vertical. Por isso decorar "justify = horizontal" falha: o certo é "justify = eixo principal, seja ele qual for".

03

As propriedades dos itens

flex-grow · flex-shrink · flex-basis

O container manda na distribuição geral, mas cada item pode negociar seu próprio tamanho com três propriedades — quase sempre escritas juntas no atalho flex:

PropriedadePergunta que respondePadrão
flex-growSobrou espaço — quanto eu cresço?0 (não cresce)
flex-shrinkFaltou espaço — quanto eu encolho?1 (encolhe)
flex-basisQual meu tamanho inicial antes de crescer/encolher?auto (do conteúdo)
o atalho flex 
.item { flex: 1; }            /* = grow 1, shrink 1, basis 0 → divide igual */
.item { flex: 2; }            /* cresce 2× mais que um flex: 1 */
.item { flex: 0 0 200px; }    /* não cresce, não encolhe: 200px fixos */
.item { flex: 1 1 300px; }    /* parte de 300px e flexiona */
flex-grow em ação: 1 · 2 · 1
flex: 1
flex: 2 — o dobro do espaço livre
flex: 1

Dois ajustes finos que completam o kit dos itens:

align-self e order 
.destaque {
  align-self: flex-end;  /* este item ignora o align-items do container */
  order: -1;             /* aparece primeiro, sem mudar o HTML */
}
⚠️ order muda só o visual Leitores de tela e a navegação por Tab continuam seguindo a ordem do HTML. Use order para ajustes pontuais; se a ordem lógica do conteúdo mudou, mude o HTML.
04

Receitas prontas

os layouts de todo dia

1 — Centralizar perfeitamente (a pergunta mais feita da história do CSS):

centro absoluto 
.container {
  display: flex;
  justify-content: center;  /* centro do eixo principal */
  align-items: center;      /* centro do eixo cruzado */
  min-height: 100vh;        /* precisa de altura para centralizar verticalmente */
}

2 — Navbar: logo à esquerda, links à direita. O space-between empurra os extremos:

navbar 
.navbar {
  display: flex;
  justify-content: space-between;  /* logo ⟷ links */
  align-items: center;             /* tudo alinhado na vertical */
  padding: 0 24px;
}
.nav-links { display: flex; gap: 16px; }

3 — Cards de mesma largura que quebram linha:

grade de cards

📦 E quando usar Grid? Flexbox distribui itens em uma dimensão (uma linha ou uma coluna — mesmo com wrap, cada linha é independente). Quando você precisa alinhar em linhas e colunas ao mesmo tempo (um painel, uma galeria rígida), a ferramenta é o display: grid — próximo passo depois desta página.
05

Pratique

fixe o conteúdo

Pense na resposta antes de revelar.

1Eixos girados ◆ Fácil

Um container tem display: flex; flex-direction: column; justify-content: center;. O conteúdo fica centralizado na horizontal ou na vertical?

Na vertical. Com column, o eixo principal aponta para baixo — e justify-content sempre trabalha no eixo principal. Para centralizar na horizontal nesse caso, o papel é do align-items: center.

2Sidebar fixa + conteúdo flexível ◆ Médio

Você quer um layout com sidebar de exatamente 240px e o conteúdo ocupando todo o resto da largura. Escreva o CSS usando o atalho flex nos dois itens.

resposta_2
.layout   { display: flex; }
.sidebar  { flex: 0 0 240px; }  /* não cresce, não encolhe, 240px fixos */
.conteudo { flex: 1; }          /* absorve todo o espaço restante */
3O item que esmagou ◆ Difícil

Numa navbar flex, o logo tem width: 150px, mas em telas estreitas ele aparece espremido com uns 80px. Ninguém escreveu flex-shrink em lugar nenhum. Por que ele encolheu, e como impedir?

Porque o padrão de todo item flex é flex-shrink: 1 — quando falta espaço, todos encolhem proporcionalmente, mesmo com width definido (o width vira só o tamanho inicial). Para travar:

resposta_3
.logo { flex-shrink: 0; }      /* este item não negocia: 150px sempre */
/* ou, no atalho: */
.logo { flex: 0 0 150px; }
