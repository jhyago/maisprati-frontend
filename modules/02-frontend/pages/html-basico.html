Estrutura & Conteúdo

HTML Básico

HTML é o esqueleto de toda página: ele define o que existe — títulos, parágrafos, imagens, tabelas, formulários — e não como aparece (isso é trabalho do CSS). Aqui você monta documentos do zero, identifica elementos com id e class, e estrutura dados em tabelas e formulários.

01

Anatomia de um documento

a moldura de toda página

Toda página HTML segue o mesmo molde. O navegador lê de cima para baixo: primeiro o <!DOCTYPE html> (diz "isto é HTML5"), depois o <head> com informações sobre a página (que não aparecem na tela), e por fim o <body> com o conteúdo visível.

index.html 
<!DOCTYPE html>
<html lang="pt-BR">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Minha primeira página</title>
  </head>
  <body>
    <h1>Olá, mundo!</h1>
    <p>Meu primeiro parágrafo em HTML.</p>
  </body>
</html>
resultado no navegador

Olá, mundo!

Meu primeiro parágrafo em HTML.

<head>
Metadados: título, idioma, codificação, links de CSS. Não aparece na página.
<body>
Tudo o que o usuário vê: textos, imagens, botões, formulários.
🔑 Tag, elemento e atributo Uma tag é a marcação: <p> abre, </p> fecha. O elemento é a tag de abertura + conteúdo + fechamento. Um atributo é uma configuração dentro da tag de abertura, no formato nome="valor" — como lang="pt-BR" ou href="...".
02

Tags de conteúdo essenciais

texto · links · imagens · listas

Com um punhado de tags você já monta a maior parte de uma página. Cada uma carrega um significado — use a que descreve o conteúdo, não a que "parece bonita".

03

id e class: dando nome aos elementos

ganchos para CSS e JS

id e class são atributos globais — funcionam em qualquer elemento. Sozinhos eles não mudam nada na tela; eles existem para você apontar para o elemento depois e estilizá-lo (CSS) ou manipulá-lo (JavaScript). Pense neles como etiquetas.

⚠️ Regras do id Um id deve ser único na página inteira, não pode ter espaços e, por convenção, não começa com número. Repetir o mesmo id em dois elementos é HTML inválido e quebra links de âncora, <label> e JavaScript.
💡 Regra prática: prefira class Use class para qualquer estilo que possa se repetir (a esmagadora maioria dos casos). Reserve o id para quando você precisa de um alvo único: o destino de um link (<a href="#aviso">), o for de um rótulo de formulário, ou um elemento que o JavaScript vai buscar com getElementById.
04

Tabelas

dados em linhas e colunas

Tabela serve para dados tabulares — uma planilha, um placar, preços. Ela não serve para fazer layout da página (isso é trabalho do CSS, no próximo tópico). A estrutura encaixa de fora para dentro: <table> contém linhas <tr>, que contêm células de cabeçalho <th> ou de dado <td>.

uma tabela completa 
<table>
  <caption>Notas da turma</caption>
  <thead>
    <tr>
      <th scope="col">Aluno</th>
      <th scope="col">Módulo</th>
      <th scope="col">Nota</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Ana</td>
      <td>HTML</td>
      <td>9,5</td>
    </tr>
    <tr>
      <td>Bruno</td>
      <td>CSS</td>
      <td>8,0</td>
    </tr>
  </tbody>
</table>

Uma tabela "crua" aparece sem bordas — quem desenha as linhas e o espaçamento é o CSS. Abaixo já aplicamos um estilo simples (border e padding) para ficar legível:

<th> vs <td>
th = cabeçalho (negrito, centralizado); td = dado comum.
scope
scope="col" ou "row" diz a quem o cabeçalho se refere — leitores de tela agradecem.
colspan / rowspan
Fazem uma célula ocupar várias colunas ou linhas.
📐 thead, tbody (e tfoot) Separar o cabeçalho (thead) do corpo (tbody) não muda a aparência, mas organiza a tabela semanticamente — facilita estilizar, ordenar e dá pistas para tecnologias assistivas. Use sempre <caption> para dar um título à tabela.
05

Formulários

coletando dados do usuário

Um formulário (<form>) reúne controles onde o usuário digita ou escolhe algo: <input>, <textarea>, <select> e <button>. A peça-chave da acessibilidade é o <label>: o for dele aponta para o id do campo — exatamente o uso de id que vimos no tópico 03.

um formulário simples 
<form>
  <label for="nome">Nome</label>
  <input type="text" id="nome" name="nome" placeholder="Seu nome" required />

  <label for="email">E-mail</label>
  <input type="email" id="email" name="email" />

  <label for="senha">Senha</label>
  <input type="password" id="senha" name="senha" />

  <label for="curso">Curso</label>
  <select id="curso" name="curso">
    <option value="fs">Full Stack</option>
    <option value="data">Dados</option>
  </select>

  <label for="msg">Mensagem</label>
  <textarea id="msg" name="msg" rows="3"></textarea>

  <label><input type="checkbox" name="termos" /> Aceito os termos</label>

  <button type="submit">Enviar</button>
</form>
resultado no navegador
O atributo type muda tudo no <input>
typeO que vira
textCampo de texto comum (o padrão)
emailTexto que valida formato de e-mail
passwordEsconde os caracteres digitados
numberSó números, com setinhas
dateSeletor de data do navegador
checkboxCaixa de marcar (múltipla escolha)
radioBolinha de escolha única (mesmo name)
submitBotão que envia o formulário
💡 Sempre conecte <label> ao campo O for do rótulo deve bater com o id do input. Assim, clicar no texto já foca o campo, e leitores de tela anunciam o nome correto. Como atalho, você pode "embrulhar" o input dentro do próprio <label> (como no "Aceito os termos" acima).
🏷️ name vs id num formulário O id é o gancho de CSS/JS e o alvo do label dentro da página. O name é a chave com que o dado é enviado ao servidor (vira nome=Ana&email=...). Sem name, o campo não é enviado. Use os dois.

Pratique

fixe o conteúdo

Pense na resposta antes de revelar. Escreva o HTML num arquivo e abra no navegador.

1id ou class? ◆ Fácil

Você tem 5 botões de "comprar" iguais e 1 cabeçalho de página que aparece uma vez só. O que recebe class e o que recebe id? Por quê?

Os 5 botões recebem a mesma class="comprar" — é um estilo que se repete. O cabeçalho recebe um id="topo" — é único e pode virar destino de um link <a href="#topo">. Regra: repetiu, é class; é único, pode ser id.

3Rótulo que não clica ◆ Médio

Um colega escreveu <label for="email">E-mail</label> seguido de <input type="email" id="e-mail" />, e clicar no rótulo não foca o campo. Qual o erro?

O for="email" não bate com o id="e-mail" (um tem hífen, o outro não). Eles precisam ser idênticos. Corrija para id="email" — e lembre que id não aceita espaços e diferencia maiúsculas de minúsculas.
