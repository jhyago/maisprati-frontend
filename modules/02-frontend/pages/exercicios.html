Exercícios — Frontend

24 desafios para fixar o módulo inteiro: HTML & CSS, DOM & BOM, Promises e Consumo de APIs / Web Components. Tente resolver de cabeça antes de revelar a resposta — alguns trazem um editor para você rodar o código na hora.

HTML & CSS

6 exercícios
1id ou class? ◆ Fácil

Você tem vários botões que precisam do mesmo estilo, e um cabeçalho único que será alvo de um link âncora. Qual atributo usar em cada caso?

class nos botões (repetível, agrupa vários elementos) e id no cabeçalho (único na página, serve de alvo para #ancora). Regra: id é único; class se repete.

2Quem vence a especificidade? ◆ Médio

HTML: <p class="texto" id="intro">Oi</p>. CSS: p { color: black }, .texto { color: blue } e #intro { color: red }. De que cor fica o texto?

Vermelho. Ordem de especificidade: #id > .classe > elemento. O #intro ganha, independentemente da ordem no arquivo.

3A caixa estourou ◆ Médio

Um card com width: 300px e padding: 20px passou a ocupar 340px e quebrou o layout. Como mantê-lo em 300px reais sem remover o padding?

resposta
.card { width: 300px; padding: 20px; box-sizing: border-box; }
/* border-box faz o width JÁ incluir padding e borda */
4Centralizar com Flexbox ◆ Médio

Escreva o CSS para centralizar um filho horizontal e verticalmente dentro de um container usando Flexbox.

resposta
.container {
  display: flex;
  justify-content: center;  /* eixo principal (horizontal) */
  align-items: center;      /* eixo cruzado (vertical) */
}
5em vs rem ◆ Difícil

Qual a diferença entre 1em e 1rem? Em que situação em pode causar um efeito "bola de neve" indesejado?

rem é sempre relativo à fonte da raiz (<html>) — estável. em é relativo à fonte do próprio elemento/pai. Aninhando elementos com font-size: 1.2em, cada nível multiplica o anterior (1.2 → 1.44 → 1.728…), crescendo descontroladamente. Para tamanhos previsíveis, prefira rem.

6O selo no canto ◆ Difícil

Um selo position: absolute deveria ficar no canto de um card, mas foi parar no canto da página. Por quê, e qual a correção?

O absolute se ancora no ancestral posicionado mais próximo; sem nenhum, usa a página. Dê position: relative ao card (o pai) para torná-lo a referência:

resposta
.card { position: relative; }
.selo { position: absolute; top: -10px; right: -10px; }

DOM & BOM

6 exercícios
7Selecionar o primeiro ◆ Fácil

Qual método pega o primeiro elemento com a classe ativo? E qual pega todos?

document.querySelector('.ativo') pega o primeiro; document.querySelectorAll('.ativo') pega todos (uma NodeList).

8textContent ou innerHTML? ◆ Fácil

Vai exibir um comentário digitado por um usuário. Qual propriedade usar, e por quê?

textContent — trata o valor como texto puro, então um <script> escondido não roda. innerHTML com dado de usuário abre brecha de XSS.

9Marque os pares ◆ Médio

Complete: para cada número de 0 a 5, registre no console se o índice é "par" ou "ímpar" usando forEach.

solução
[0,1,2,3,4,5].forEach((n, i) => {
  console.log(`${i}: ${i % 2 === 0 ? 'par' : 'ímpar'}`);
});
10Criar e anexar um nó ◆ Médio

Escreva as três linhas que criam um <li> com o texto "Novo" e o anexam como último filho de uma <ul> guardada em lista.

resposta
const li = document.createElement('li');  // 1) cria
li.textContent = 'Novo';                   // 2) configura
lista.append(li);                          // 3) anexa
11Delegação de eventos ◆ Difícil

Você tem uma lista que ganha novos <li> dinamicamente e quer reagir a cliques em qualquer um deles, inclusive nos criados depois. Por que pôr um listener em cada <li> é ruim, e qual a alternativa?

Listeners individuais não pegam os <li> futuros e desperdiçam memória. Use delegação: um único listener no <ul> pai e identifique o alvo por event.target (graças ao bubbling).

resposta
ul.addEventListener('click', (e) => {
  if (e.target.tagName === 'LI') console.log('clicou em', e.target.textContent);
});
12Guardar um objeto ◆ Difícil

Salve { nome: 'Ana', favoritos: [1, 2] } no localStorage e leia de volta como objeto. Por que não dá para guardar o objeto diretamente?

O localStorage só guarda strings. Serialize com JSON.stringify ao gravar e JSON.parse ao ler:

resposta
localStorage.setItem('u', JSON.stringify({ nome: 'Ana', favoritos: [1, 2] }));
const u = JSON.parse(localStorage.getItem('u'));

Promises & Async

6 exercícios
13Os três estados ◆ Fácil

Quais são os três estados possíveis de uma Promise, e o que significa "settled"?

pending (pendente), fulfilled (resolvida) e rejected (rejeitada). "Settled" = já se decidiu (fulfilled ou rejected) — e esse estado final é imutável.

14Adivinhe a ordem ◆ Médio

O que imprime? Rode mentalmente e depois confira no editor.

A, C, B. Os logs síncronos saem primeiro; o .then é uma microtarefa que só roda quando a thread principal esvazia.

15.then → async/await ◆ Médio

Reescreva fetchDados().then(d => console.log(d)).catch(e => console.log(e)) usando async/await com try/catch.

resposta
async function run() {
  try {
    const d = await fetchDados();
    console.log(d);
  } catch (e) {
    console.log(e);
  }
}
16Paralelo vs sequencial ◆ Difícil

Três requisições independentes levam 300ms, 500ms e 200ms. Quanto tempo leva o conjunto se você usar dois await seguidos vs Promise.all? Qual escolher?

await em sequência soma: 300+500+200 = ~1000ms. Promise.all dispara tudo junto: leva o tempo da mais lenta, ~500ms. Para tarefas independentes, use Promise.all.

resposta
const [a, b, c] = await Promise.all([req1(), req2(), req3()]);
17all que não pode falhar ◆ Difícil

Você quer o resultado de 4 Promises mesmo que algumas rejeitem, sem uma falha derrubar as outras. Qual combinador, e qual o formato do retorno?

Promise.allSettled. Devolve um array de objetos: { status: 'fulfilled', value } ou { status: 'rejected', reason } para cada Promise — nenhuma falha interrompe as demais.

18Promise com timeout ◆ Difícil

Escreva uma função comTimeout(promise, ms) que resolve com a promise se ela terminar a tempo, ou rejeita com "timeout" se estourar o tempo. Dica: Promise.race.

resposta
function comTimeout(promise, ms) {
  const limite = new Promise((_, reject) =>
    setTimeout(() => reject(new Error('timeout')), ms)
  );
  return Promise.race([promise, limite]);  // o que terminar primeiro vence
}

APIs & Web Components

6 exercícios
19Os dois await do fetch ◆ Fácil

Por que const r = await fetch(url); const d = await r.json(); precisa de dois await?

O primeiro espera a resposta chegar (status, cabeçalhos); o segundo espera o corpo ser lido e convertido de JSON para objeto — r.json() também é assíncrono.

20O 404 silencioso ◆ Médio

"Meu try/catch em volta do fetch pega qualquer erro da API." Verdadeiro ou falso? Justifique.

Falso. O fetch só rejeita em falha de rede. Um 404/500 é uma resposta "bem-sucedida" para ele — cheque resposta.ok e dê throw manualmente.

21A pegadinha do ViaCEP ◆ Médio

Você busca 99999-999 no ViaCEP. Vem status 200 e resposta.ok === true, mas o CEP não existe. Como detectar esse caso?

O ViaCEP devolve 200 com corpo { "erro": true } para CEP inexistente. Inspecione o corpo:

resposta
const dados = await resposta.json();
if (dados.erro) throw new Error('CEP não encontrado');
22Montar um POST ◆ Médio

Escreva um fetch que envie { nome: 'Ana' } por POST para /usuarios como JSON. Não esqueça o cabeçalho.

resposta
await fetch('/usuarios', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ nome: 'Ana' }),
});
23O nome inválido ◆ Fácil

Por que customElements.define('counter', Counter) dá erro, mas 'click-counter' funciona?

O nome de um Custom Element precisa ter um hífen, para nunca colidir com tags nativas (que não têm). counter é inválido.

24CSS que vazou ◆ Difícil

O p { color: red } dentro de um Web Component está afetando a página inteira. O que faltou, e como o Shadow DOM resolve?

Faltou o Shadow DOM. Sem attachShadow, o <style> vive no DOM normal e vaza. Renderize dentro da árvore-sombra para isolar o CSS:

resposta
constructor() { super(); this.attachShadow({ mode: 'open' }); }
connectedCallback() {
  this.shadowRoot.innerHTML = `<style>p { color: red }</style><p>…</p>`;
}
