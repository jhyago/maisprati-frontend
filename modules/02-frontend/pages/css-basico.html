Aparência & Layout

CSS Básico

Se o HTML é o esqueleto, o CSS é a roupa. Com seletores você aponta para os elementos (usando aquelas etiquetas id e class) e, com propriedades, decide cor, espaçamento e posição. Aqui você entende o box model e domina os cinco tipos de position.

01

Como aplicar CSS & anatomia de uma regra

seletor { propriedade: valor }

Uma regra de CSS tem sempre a mesma forma: um seletor (quem vou estilizar) seguido de um bloco com declarações entre chaves. Cada declaração é um par propriedade: valor;.

anatomia 
seletor {
  propriedade: valor;   /* uma declaração */
  color: #0062c0;       /* propriedade "color", valor "#0062c0" */
  font-size: 18px;
}

Existem três lugares para colocar CSS — em ordem do pior para o melhor:

as três formas 
<!-- 1) Inline: no próprio elemento. Evite — não reaproveita. -->
<p style="color: blue;">Texto azul</p>

<!-- 2) Interno: no <head>, dentro de <style>. Serve uma página só. -->
<style>
  p { color: blue; }
</style>

<!-- 3) Externo: arquivo .css separado. O jeito recomendado. -->
<link rel="stylesheet" href="estilos.css" />
💡 Prefira o CSS externo Um arquivo .css separado mantém estrutura (HTML) e estilo (CSS) desacoplados, é reaproveitado por todas as páginas e fica em cache no navegador. CSS inline só para um ajuste pontual; nunca como hábito.

⚖️ Especificidade: quem ganha quando dois seletores brigam Se duas regras mexem na mesma propriedade do mesmo elemento, vence a mais específica: #id > .classe > elemento. Empatou? Vence a que vem por último no arquivo. Fuja do !important — ele atropela tudo e vira uma dor de cabeça para manter.
03

O box model

content · padding · border · margin

No CSS, todo elemento é uma caixa retangular. Essa caixa tem quatro camadas, de dentro para fora. Entender essas camadas resolve 90% das dúvidas de "por que tem esse espaço aqui?".

margin — espaço POR FORA, separa de outros elementos
border — a borda
padding — espaço interno
content — o conteúdo (texto, imagem)
content
o conteúdo em si
padding
respiro interno, dentro da borda
border
a linha da borda
margin
distância para os vizinhos
controlando a caixa 
.card {
  padding: 16px;            /* respiro interno nos 4 lados */
  border: 2px solid #0062c0;
  margin: 24px;             /* afasta dos vizinhos */
  margin: 10px 20px;        /* atalho: 10 em cima/baixo, 20 nas laterais */
}
⚠️ A pegadinha clássica: box-sizing Por padrão (content-box), width mede só o conteúdo — somar padding e border deixa a caixa maior que o esperado. A solução que todo projeto adota: box-sizing: border-box; faz o width já incluir padding e borda. Os dois quadrados abaixo têm width: 200px e o mesmo padding:
content-box (padrão) vs border-box
content-box → ocupa 246px reais
border-box → ocupa 200px reais

05

Posicionamento

static · relative · absolute · fixed · sticky

Por padrão, todo elemento é position: static — ele fica no fluxo normal, um depois do outro. Mudar a position libera as propriedades top, right, bottom e left para deslocar a caixa. Brinque com o visualizador abaixo — role a moldura para sentir a diferença entre absolute e sticky:

positionSai do fluxo?Referência de top/leftUso típico
staticNão (padrão)— (ignora offsets)O comportamento normal
relativeNão (guarda o lugar)A própria posição originalAjuste fino + virar âncora do absolute
absoluteSimAncestral posicionado mais próximoSelo "novo", tooltip, badge sobre um card
fixedSimA janela do navegadorCabeçalho/botão que não rola
stickyNão, até "grudar"O container que rolaCabeçalho de seção que fixa ao rolar
💡 O combo mais usado: relative + absolute Um elemento absolute se ancora no ancestral posicionado mais próximo. Então o padrão é: dê position: relative ao pai (sem offsets, só para virar referência) e position: absolute ao filho. Assim o filho se posiciona dentro do pai, e não na página inteira. É exatamente como se prega um selo "NOVO" no canto de um card:
relative (pai) + absolute (selo)
NOVO Plano Pro

O selo está ancorado no canto deste card.

⚠️ z-index e a regra do sticky O z-index controla quem fica na frente quando caixas se sobrepõem — mas só funciona em elementos posicionados (qualquer position diferente de static). E sticky só "gruda" se você definir um lado (top: 0, por exemplo) e o elemento estiver dentro de um container que rola.

Pratique

Pense na resposta antes de revelar.

1Quem vence? ◆ Fácil

O HTML é <p class="texto" id="intro">Olá</p>. O CSS tem .texto { color: green; } e #intro { color: red; }. De que cor fica o texto?

Vermelho. O seletor de #id é mais específico que o de .classe, então #intro ganha, independentemente da ordem no arquivo.

2A caixa "estourou" ◆ Médio

Você definiu width: 300px num card, adicionou padding: 20px e ele passou a ocupar 340px, quebrando o layout. Como fazer o card respeitar os 300px sem tirar o padding?

resposta_2
.card {
  width: 300px;
  padding: 20px;
  box-sizing: border-box;   /* agora os 300px JÁ incluem o padding */
}
3Selo no canto ◆ Difícil

Você quer um selo "-50%" colado no canto superior direito de um card. Coloca o selo como position: absolute, mas ele vai parar no canto da página, não do card. Por quê, e como resolver?

O absolute se ancora no ancestral posicionado mais próximo; sem nenhum, ele usa a página. A correção é dar position: relative ao card (o pai), tornando-o a referência:

resposta_3
.card { position: relative; }                 /* vira a referência */
.selo { position: absolute; top: -10px; right: -10px; }
